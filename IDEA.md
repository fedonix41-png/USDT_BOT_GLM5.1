### ❌ Что мы убиваем (Антипаттерны чат-ботов)
1. **Слепой ввод сумм через FSM.** Сейчас клиент вводит сумму, ждёт ответа бота, потом узнает итог. Это медленно и вызывает трение.
2. **Бесконечные списки заявок.** Операторы смотрят ленту сообщений. Это неудобно для модерации и приводит к потере контекста.
3. **Ручной ввод Telegram ID.** Админы назначают роли по ID. В 2026 году это моветон.
4. **Текстовые подтверждения.** «Вы уверены?» — нет, нужны свайпы или bottom sheets.

### 💎 UX-Архитектура: Клиентский интерфейс (Web3 Wallet Experience)

**Визуальный стиль:** Глубокий тёмный космос (OLED Black / Slate 900), Glassmorphism (размытые подложки карточек), акцентные неоновые цвета (Tether Green `#26A17B` для покупки, Rose/Red для продажи). Обязательное использование `Telegram.WebApp.HapticFeedback` на каждое действие.

#### 1. Главный экран (Dashboard) вместо меню
* **Top Bar:** Баланс (если планируется), статус верификации или просто приветствие с аватаркой.
* **Центральный блок (Action Cards):** Две массивные, красивые кнопки/карточки: **Buy USDT** и **Sell USDT**. Если `buy_enabled=False` (из твоего Redis кэша) — карточка становится полупрозрачной (Disabled) с красивым бейджем "Paused".
* **Сводка курсов:** Стеклянная плашка, показывающая текущие курсы в реальном времени.

#### 2. Флоу обмена (Smart Calculator)
* **Как сейчас:** `/buy` -> введи сумму -> проверка.
* **TMA Парадигма:** Двухсторонний калькулятор (как в Uniswap или 1inch).
  * Поле 1: «Вы отдаете (RUB)»
  * Поле 2: «Вы получаете (USDT)»
  * **UX фишка:** Клиент вводит данные в любое поле, второе пересчитывается моментально на клиенте (курс подтягивается по API).
  * Валидация (от 0.01 до 100 000) происходит *до* отправки запроса на сервер — кнопка "Exchange" просто неактивна, пока есть ошибка (с красной подсказкой).
* **Confirm:** По нажатию открывается нативный Telegram Bottom Sheet (всплывающая шторка снизу) с итоговой информацией и свайпом для подтверждения (Slide to Confirm).

#### 3. Состояние заявки (Active Order Tracker)
* Переход на страницу "Заявка #123".
* Огромный QR-код (опционально) или красиво оформленные реквизиты (`payment_link`).
* Кнопка "Скопировать реквизиты" с микроанимацией галочки (Success).
* **Статус-бар:** Прогресс-бар из 3 шагов: *Ожидает оплаты ➝ В обработке ➝ Завершено*. (Апдейт через WebSocket или long-polling твоего API).
* Кнопка отмены — ненавязчивая, в самом низу. Жалоба на битую ссылку («Ссылка не работает») прячется в меню "три точки" в правом верхнем углу.

### 📊 UX-Архитектура: Операторский Backoffice (Mobile CRM)

Для операторов и админов мы делаем интерфейс плотным (data-rich), но удобным для пальцев.

#### 1. Управление заявками (Order Management)
* **Как сейчас:** Сообщения с кнопками [✅ Завершить] [❌ Отменить].
* **TMA Парадигма:** Таблица-список (Data Grid) с табами: `New | Processing | Completed`.
* **Карточка заявки:** Компактная. Показывает тип (Buy/Sell), сумму, время с момента создания (таймер).
* **Быстрые действия:** Свайп карточки вправо — "Approve/Завершить" (зеленый фон). Свайп влево — "Cancel" (красный фон). Это экономит клики оператору при массовой обработке.

#### 2. Админ-Дашборд (God Mode)
* Сетка с KPI (из твоего `OrderService.get_statistics`): Объем торгов за 24ч, активные заявки, профит.
* **Quick Toggles:** Переключатели в стиле iOS (Toggle Switches) для `bot_enabled`, `buy_enabled`, `sell_enabled`. Админ нажимает -> переключатель анимируется -> идёт запрос к API -> обновляется `global_settings`. Мгновенная визуальная обратная связь.
* **Управление курсами:** Слайдеры или инпуты со стрелками вверх/вниз для микро-корректировок курса покупки/продажи (без необходимости каждый раз печатать цифры на клавиатуре).

#### 3. Управление ролями
* Вывод списка всех юзеров (с пагинацией/поиском).
* Клик по юзеру открывает Bottom Sheet: выпадающий список (Select) для назначения роли (Client -> Operator -> Admin). Никаких ручных вводов Telegram ID!

### 🛠 Как это вписывается в твой текущий код (Safe Integration)

Мы **ничего не ломаем** в твоём Python/Alembic/ARQ бэкенде.
1. **Telegram-бот становится роутером.** Бот по команде `/start` выдает красивую Inline-кнопку **[ 💎 Open App ]**. (Можно сделать разные кнопки для юзеров и операторов/админов, проверяя `user.role` в хэндлере).
2. **Авторизация.** Бот генерирует одноразовый токен (у тебя есть модель `APIToken` в базе!) и передает его в URL Mini App (например: `https://app.com?token=xyz`).
3. **API.** Frontend делает запросы к твоему `REST API Middleware (aiohttp)`, авторизуясь по JWT.
4. **Уведомления.** ARQ Worker продолжает отправлять сообщения в чаты! Это отлично работает как система алертов: «Поступила новая заявка! [Открыть админку] (ссылка на TMA)».


1. **Дизайн-система (`webapp/src/index.css`)**
   - цветовая палитра: глубокий `OLED Black` (`#0B0E14`) для фона, `Tether Green` и насыщенный `Red` для акцентов с легким неоновым свечением.
   - Разработаны утилиты Glassmorphism (`.glass-panel`) — полупрозрачные карточки с blur-эффектом и красивым бликом (`::before` градиент), который перекатывается при наведении.
   - плавные микроанимации (`fadeIn`, `scale` при нажатии), чтобы интерфейс казался "живым".
   - Использован шрифт `Inter` для идеальной читаемости финансовых данных.

2. **Главный Дашборд (`webapp/src/App.jsx`)**
   - Telegram WebApp SDK. Приложение теперь автоматически "раскрывается" (`WebApp.expand()`) и захватывает цвета темы Telegram (чтобы верхний хедер сливался с нашим фоном).
   - *Haptic Feedback* (`impactOccurred`) — при нажатии на действия телефон будет вибрировать, создавая ощущение премиальности.
   - Добавлены две огромные стеклянные карточки действий — «Buy USDT» и «Sell USDT» с градиентными иконками (библиотека `lucide-react`).
   - Блок "Current Rates" с красивой стилизацией

3. **Мобильная оптимизация (`webapp/index.html`)**
   - Изменить `meta viewport`, чтобы запретить зум (user-scalable=no). Это критически важно для Telegram WebApp, иначе при случайном двойном тапе интерфейс будет "уезжать", ломая ощущение нативности.
