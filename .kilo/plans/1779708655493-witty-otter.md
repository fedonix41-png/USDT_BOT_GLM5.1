# План исправления багов: Отмена, Смена курса, ENCRYPTION_KEY

## Корневые причины (2 независимых бага)

### Баг A: Неверный порядок middleware → `user` всегда `None` в хендлерах

**Симптомы:**
- Кнопка "❌ Отмена" не восстанавливает клавиатуру (показывает "Действие отменено." без меню)
- Смена курса показывает "Ошибка." вместо сохранения нового курса

**Причина:** В `app/bot.py:34-44` middleware зарегистрированы в неверном порядке.
Комментарий гласит: "Last registered = outermost" — это **неверно** для aiogram 3.x.

В aiogram `MiddlewareManager.wrap_middlewares` делает `reversed(middlewares)`,
поэтому **первый зарегистрированный** middleware = самый внешний (outermost),
а **последний** = самый внутренний (innermost, ближе всего к хендлеру).

Текущий код:
```python
dp.message.middleware(RoleGuardMiddleware())    # 1й → outermost
dp.message.middleware(BotStatusMiddleware())    # 2й
dp.message.middleware(UserMiddleware())         # 3й
dp.message.middleware(DBSessionMiddleware())    # 4й → innermost
```

Порядок выполнения: RoleGuard → BotStatus → **UserMiddleware** → **DBSession** → хендлер

`UserMiddleware` выполняется **ДО** `DBSessionMiddleware`, поэтому `session` ещё не в `data`,
и `UserMiddleware` пропускает загрузку пользователя → `data["user"] = None`.

Все хендлеры, зависящие от `user`, получают `None`:
- `cancel_any_fsm` → `user is None` → нет клавиатуры
- `process_new_sell_rate` / `process_new_buy_rate` → `user is None` → "Ошибка."
- `process_support_message` → `user is None` → "Ошибка. Попробуйте снова."

**Исправление:** Поменять порядок регистрации на обратный:
```python
dp.message.middleware(DBSessionMiddleware())     # 1й → outermost
dp.message.middleware(UserMiddleware())           # 2й (нужен session)
dp.message.middleware(BotStatusMiddleware())      # 3й (нужен user)
dp.message.middleware(RoleGuardMiddleware())       # 4й → innermost (нужен user)
```

То же самое для `callback_query`.

### Баг B: ENCRYPTION_KEY в формате base64 вместо hex

**Симптомы:**
- Покупка/продажа USDT выкидывает `ValueError: ENCRYPTION_KEY must be 64 hex chars (32 bytes), got 44`
- Логи показывают ошибку в `buy.py:35` и `sell.py:35`

**Причина:** В `.env` указан `ENCRYPTION_KEY=WGr9MQKiohR02-OecabUI1tMyDkO0JNlh3NWl182yxs=`
(44 символа, base64). А `EncryptionService.__init__` ожидает 64 hex-символа.

**Исправление:** Обновить `EncryptionService.__init__` чтобы он принимал оба формата:
- 64 hex-символа → `unhexlify()`
- base64-строка → `base64.b64decode()`
- Проверить результат: должны получиться ровно 32 байта (AES-256)
