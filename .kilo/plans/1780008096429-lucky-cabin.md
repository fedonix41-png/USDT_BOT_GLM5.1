# План реализации Production WebApp для USDT Bot

## Обзор

Реализация Telegram Mini App (TMA) с Web3-дизайном для клиентского и операторского интерфейсов. Проект использует существующий React-стек с @twa-dev/sdk и расширяет его для полноценного UX.

---

## Этап 1: Архитектурные доработки Backend (API)

### 1.1 Новый роутер `/api/v1/me` — данные текущего пользователя
**Файл:** `app/api/routers/me.py`

```python
# Endpoint: GET /api/v1/me
# Возвращает:
# - user: UserResponse (текущий пользователь)
# - settings: { buy_enabled, sell_enabled } (для disabled состояния кнопок)
# - orders: список собственных заявок (с пагинацией)
```

**Зависимости:** `get_current_user`, `SettingsService`
**Auth:** JWT токен в Authorization header (полученный через одноразовый токен в URL)

### 1.2 Расширение `/api/v1/orders` — создание заявки через WebApp
**Файл:** `app/api/routers/orders.py`

```python
# POST /api/v1/orders
# Payload: { order_type: "buy" | "sell", amount_usdt: decimal }
# Возвращает: OrderResponse с decrypted_payment_link

# GET /api/v1/orders/my
# Возвращает: список заявок текущего пользователя (для Dashboard)
```

**Логика:**
- Валидация amount (0.01 - 100,000) на сервере
- Получение текущего курса через `RateService`
- Генерация payment_link через `SettingsService.get_payment_link()`
- Создание заявки через `OrderService.create_order()`
- Возврат payment_link в расшифрованном виде для WebApp

**Отличие от текущего:** WebApp вызывает API напрямую вместо WebApp.sendData → bot

### 1.3 WebSocket для real-time обновлений заявок
**Файл:** `app/api/routers/ws.py`

```python
# WebSocket endpoint: /api/v1/ws/orders/{order_id}
# Push статус: created → processing → completed/cancelled
```

**Альтернатива:** Long-polling каждые 5 секунд через `GET /api/v1/orders/{id}` (fallback)

## Этап 5: Интеграция с Telegram Bot

### 5.1 Обновление `/start` хендлера
**Файл:** `app/handlers/start.py`

```python
# Генерация одноразового токена
# URL: https://webapp.com?token={one_time_token}
# Токен привязывается к пользователю и имеет TTL=5 минут
```

**Новые элементы:**
- Inline-кнопка **[ 💎 Open App ]** вместо reply-кнопок
- Разные кнопки для ролей (Client vs Operator/Admin)
- Токен генерируется через `APITokenRepository.create_one_time_token()`

**Новый эндпоинт:** `/api/v1/auth/webapp` — обменивает одноразовый токен на JWT

**Логика:**
- GET `/api/v1/auth/webapp?token=xyz`
- Проверка токена в базе (`api_tokens` с полем `one_time=True`)
- Возврат JWT access_token + refresh_token
- Токен помечается как использованный (revoked=True)

**Миграция:** `migrations/versions/005_add_one_time_to_api_token.py` — добавить `one_time: Mapped[bool]` поле

## Риски и решения

| Риск | Решение |
|------|---------|
| Telegram WebApp API ограничен на desktop | Дизайн адаптивный, fallback для desktop |
| WebSocket недоступен | Long-polling каждые 5 секунд |
| HapticFeedback не работает на старых версиях | try/catch с фоллбеком |
| Безопасность токенов в URL | Одноразовые токены TTL=5 мин, HTTPS обязательно |
| Нет статуса "processing" в БД | Использовать `created` для "New", добавить колонку `status_changed_at` или добавить `processing` enum в будущей миграции |
