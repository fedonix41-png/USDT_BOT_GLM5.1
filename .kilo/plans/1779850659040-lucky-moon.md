### 1.3. Repository — `app/repositories/user_repo.py`

### 1.4. Service — `app/services/user_service.py`
Добавить методы:
```python
async def block_user(self, user_id: int, blocked_by_user_id: int) -> User | None:
    user = await self.user_repo.set_blocked(user_id, True)
    if user:
        await self.audit_repo.log(
            user_id=blocked_by_user_id,
            action="ban_user",
            details={"target_user_id": user_id},
        )
    return user

async def unblock_user(self, user_id: int, unblocked_by_user_id: int) -> User | None:
    user = await self.user_repo.set_blocked(user_id, False)
    if user:
        await self.audit_repo.log(
            user_id=unblocked_by_user_id,
            action="unban_user",
            details={"target_user_id": user_id},
        )
    return user
```

### 1.5. Handler — новый файл `app/handlers/admin/ban_user.py`
Следуя паттерну `assign_roles.py`:

**Callback `mgmt:ban_user`** (в `management.py`):
- Проверка роли: admin/super_admin
- Запуск FSM `BanUserStates.waiting_target_user`
- Сообщение: «Введите Telegram ID пользователя для блокировки:»

**Обработчик `BanUserStates.waiting_target_user`:**
- Парсинг Telegram ID
- Проверка: пользователь существует в БД
- Проверка: целевой пользователь имеет роль `client` или `operator` (не admin/super_admin)
- Проверка: целевой пользователь ещё не заблокирован
- Вызов `user_service.block_user()`
- Уведомление заблокированному пользователю: «🚫 Вы заблокированы в боте обмена USDT.»
- Уведомление админу: «✅ Пользователь @username заблокирован.»
- Уведомление в чаты через NotificationService

**Callback `mgmt:unban_user`** (в `management.py`):
- Аналогично, запуск FSM `UnbanUserStates.waiting_target_user`

**Обработчик `UnbanUserStates.waiting_target_user`:**
- Парсинг Telegram ID
- Проверка: пользователь существует и заблокирован
- Вызов `user_service.unblock_user()`
- Уведомление разбаненному: «✅ Вы разблокированы в боте обмена USDT.»
- Уведомление админу: «✅ Пользователь @username разблокирован.»
