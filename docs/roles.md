# Ролевая модель

> FSM-детали сценариев — см. `scenarios.md`. Middleware проверки — см. `architecture.md#middleware`.

---

## Иерархия ролей

```
super_admin  (уровень 4) — максимальные права
    │
    ├── admin  (уровень 3)
    │       │
    │       ├── operator  (уровень 2)
    │       │       │
    │       │       └── client  (уровень 1) — базовая роль
```

**Правило наследования:** каждая роль включает все права нижестоящих ролей, плюс собственные уникальные права.

**Реализация в коде:** `_ROLE_HIERARCHY` в `app/middlewares/role_guard.py`:

```python
_ROLE_HIERARCHY: dict[RoleEnum, int] = {
    RoleEnum.client: 0,
    RoleEnum.operator: 1,
    RoleEnum.admin: 2,
    RoleEnum.super_admin: 3,
}
```

## Меню по ролям

### Client (Клиент)

Базовая роль. Все новые пользователи получают её при `/start`.

```
[💰 Купить] [💸 Продать]
[📊 Курсы] [📞 Поддержка]
```

**Ограничения:**
- `buy_enabled=False` → кнопка «Купить» заменяется на «🛑 Закуп остановлен»
- `sell_enabled=False` → кнопка «Продать» заменяется на «🛑 Продажа остановлена»
- `bot_enabled=False` → все клиентские действия блокируются с сообщением «Бот временно недоступен»

---

### Operator (Оператор)

Роль для обработки клиентских заявок. Не может создавать заявки от своего имени.

```
[📋 Заявки] [📊 Курсы]
[📈 Статистика]
```

**Особенности:**
- Видит заявки всех клиентов
- Работает даже когда бот отключён для клиентов (`bot_enabled=False`)

---

### Admin (Администратор)

Полный управленческий доступ, кроме назначения других админов.

**Reply-клавиатура:**

```
[📋 Заявки] [📈 Статистика] [📊 Курсы]
[⚙️ Управление]
```

**Inline-панель управления:**

```
[🔄 Курс покупки] [🔄 Курс продажи]
[🔗 Реквизиты]
[✅ Закуп вкл / ⏸ Стоп закуп] [✅ Продажа вкл / ⏸ Стоп продажа]
[🛑 Отключить бота / ✅ Включить бота]
[➕ Чаты] [👤 Оператор]
[🚫 Забанить] [✅ Разбанить]
[🔙 Закрыть]
```

**Особенности:**
- Все действия логируются в `audit_logs`
- При смене реквизитов — автоматическое обновление сообщений клиентов с битой ссылкой (через ARQ)
- Кнопки-переключатели обновляются inline (без перерисовки всей клавиатуры)

---

### SuperAdmin (Суперадминистратор)

Максимальные права. Единственный, кто может назначать админов.

**Reply-клавиатура:** такая же, как у Admin.

**Inline-панель управления:**

```
[🔄 Курс покупки] [🔄 Курс продажи]
[🔗 Реквизиты]
[✅ Закуп вкл / ⏸ Стоп закуп] [✅ Продажа вкл / ⏸ Стоп продажа]
[🛑 Отключить бота / ✅ Включить бота]
[➕ Чаты] [👤 Оператор]
[👑 Админ]                                          ← только у SuperAdmin
[🚫 Забанить] [✅ Разбанить]
[🔙 Закрыть]
```

**Особенности:**
- Назначается при первом `/start` через `SUPER_ADMIN_TELEGRAM_ID` из `.env`
- Единственная роль, которую невозможно назначить через интерфейс бота
- Кнопка `👑 Админ` видна только SuperAdmin

## Механизмы проверки прав

### Слой 1: RoleFilter + RoleGuardMiddleware

Фильтр на уровне роутеров — проверяет числовые уровни ролей:

```python
class RoleFilter:
    def __init__(self, min_role: RoleEnum) -> None:
        self.min_role = min_role
        self.min_level = _ROLE_HIERARCHY[min_role]

    async def __call__(self, message: TelegramObject, data: dict[str, Any]) -> bool:
        user: User | None = data.get("user")
        if user is None:
            return False
        user_level = _ROLE_HIERARCHY.get(user.role, 0)
        return user_level >= self.min_level
```

### Слой 2: Прямые проверки в хэндлерах

Защита от IDOR — каждый обработчик проверяет `user.role` явно:

```python
@router.callback_query(F.data == "mgmt:rate_buy")
async def start_change_buy_rate(callback: CallbackQuery, state: FSMContext,
                                 session: AsyncSession, user: User | None) -> None:
    if user is None or user.role not in (RoleEnum.admin, RoleEnum.super_admin):
        await callback.answer("У вас нет прав.", show_alert=True)
        return
```

### Слой 3: API role checking

REST API проверяет роли через `require_min_role()` в `app/api/deps.py`.

### TelePay: новые API-эндпоинты и права

> **TelePay Integration** — добавленные эндпоинты и их требования к ролям.

| Эндпоинт | Метод | Мин. роль | Ограничения |
|----------|-------|-----------|-------------|
| `/api/v1/orders` | POST | client | Создание заявки через webapp |
| `/api/v1/user/orders` | GET | client | Только свои заявки |
| `/api/v1/orders/{id}/complain` | POST | client | Только своя заявка |
| `/api/v1/orders/{id}/status` | PATCH | client | Только своя заявка, только → `cancelled` |
| `/api/v1/exchange/settings` | GET | client | Комбинированные настройки (курсы, флаги, реквизиты) |
| `/api/v1/exchange/settings` | PATCH | admin | Обновление настроек обмена |

## Сообщения при нарушении прав

| Ситуация | Сообщение |
|----------|-----------|
| Клиент использует команду оператора | «У вас нет прав для этого действия.» |
| Клиент обращается при отключённом боте | «Бот временно недоступен.» |
| Покупка при остановленном закупе | «Покупка USDT временно приостановлена.» |
| Продажа при остановленной продаже | «Продажа USDT временно приостановлена.» |
| Админ пытается назначить Админа | «У вас нет прав для этого действия.» |
| Назначение роли несуществующему пользователю | «Пользователь не найден. Он должен сначала запустить бота (/start).» |
